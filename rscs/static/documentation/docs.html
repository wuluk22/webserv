Webserv Project Documentation

Overview

Webserv is a server-side application designed to handle HTTP requests and responses. Its modular architecture provides flexibility for configuration, error handling, logging, and CGI integration.

Key Components

  • CGI: Handles Common Gateway Interface-related operations, including request execution (Cgi.cpp, Cgi.hpp).
  • DirectoryHandler: Manages directory operations like browsing and serving directory content (DirectoryHandler.cpp, DirectoryHandler.hpp).
  • ErrorHandler: Centralized error reporting and response generation (ErrorHandler.cpp, ErrorHandler.hpp).
  • HttpRequestHandler: Parses and processes HTTP requests, delegates to utility functions (HttpRequestHandler.*).
  • HttpResponseHandler: Generates HTTP responses based on the request and server state (HttpResponseHandler.*).
  • Logger: Implements logging functionality for debugging and monitoring (Logger.cpp, Logger.hpp).
  • Configuration Parsing: Handles server configuration parsing:
    • ConfigParser: Parses the configuration file.
    • ServerConfig: Represents server and location block configurations.
    • Utils: Provides utilities like path validation and token counting.
